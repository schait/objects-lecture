$ dmget wb-objects --demo

Intro#

A Fruit Stand#

Say you are running a fruit stand and you want to keep track of your inventory.

Fruit

Number

Apple

6

Berry

12

Cherry

8

What data structure could we use for this?

We could use parallel arrays:

const fruits = ['apple', 'berry', 'cherry'];
const fruitCounts = [6, 12, 8];

  • How would we get the number of cherries?

  • We’d have to find the index of 'cherry', then get the value at that index in fruitCounts

Wouldn’t it be nice if we could just get the value at 'cherry'?

console.log(fruits['cherry']);  // 8

Objects#

A collection of key-value pairs

Arrays

const data = [
  value1,
  value2,
  value3
];

Objects

const data = {
  key1: value1,
  key2: value2,
  key3: value3
};

Objects in other languages

In other programming languages, objects can be called other things: dictionaries, maps, hash maps, etc. (In fact, “objects” in other languages may refer to something different). If you explore other languages, you’ll find out that most of them have a way to store collections of key-value pairs.

How objects differ from arrays:

  • Look up values using keys instead of numeric indexes

    • “Get the item stored at 'cherry'” instead of “get the third item”

  • Lookup time is very fast

    • We can find out if 'cherry' is in the object right away; arrays have to loop through each item, one-by-one

Therefore, let’s use an object to store our fruit inventory.

Objects#

Creating Objects#

  • Objects are declared using curly braces {}.

    const fruits = {};

  • Values can be any data type. Keys must be strings. Although when creating objects, you can leave off the quotes.

    // These are equivalent!

const fruits = { 'apple': 6, 'berry': 12, 'cherry': 8 };

const fruits = { apple: 6, berry: 12, cherry: 8 };

  • Keys are also referred to as properties.

Accessing values#

The most common way to access values in an object is by using the dot (.) operator:

console.log(fruits.apple);  // 6
console.log(fruits.berry);  // 12

You can also access values with bracket notation.

console.log(fruits['apple']);  // 6

When to use bracket notation#

  • The most common use-case for bracket notation is when the key name is stored in a variable:

    const fruitName = 'apple';
console.log(fruits[fruitName]); // 6

  • Note that fruits.fruitName will NOT work here!

    • This will look for a key called 'fruitName' — it won’t treat fruitName as a variable.

You must also use bracket notation instead of dot notation if the key name contains a space:

const weirdFruits = {
  'star apple': 2,
  kumquat: 4,
  loganberry: 7,
};

console.log(weirdFruits['star apple']); // 2

Adding/updating values#

  • The same syntax is used to add/update values in an object.

  • If the key exists, the value is updated.

    const fruits = {apple: 6, berry: 12, cherry: 8};

fruits.apple = 5;
console.log(fruits); // { apple: 5, berry: 12, cherry: 8}

  • If the key doesn’t exist, a new key/value pair is added.

    fruits.dragonfruit = 4;
console.log(fruits); // { apple: 5, berry: 12, cherry: 8, dragonfruit: 4 }

You can also add/update objects with bracket notation:

const weirdFruits = {'star apple': 2, kumquat: 4, loganberry: 7};

fruitsInventory['kumquat'] = 5;
// value of 'kumquat' updated to 5

weirdFruits['prickly pear'] = 3;
console.log(weirdFruits);
// {'star apple': 2, kumquat: 5, loganberry: 7, 'prickly pear': 3}

The delete operator#

The delete operator removes a key-value pair from an object.

const fruitsInventory = { apple: 6, berry: 12, cherry: 8, dragonfruit: 4 };

delete fruitsInventory.dragonfruit;
console.log(fruitsInventory);  // { apple: 6, berry: 12, cherry: 8 }

Looping through objects#

You can loop through all the keys of an object with a for...in loop:

for (const fruit in fruits) {
  console.log(fruit, fruits[fruit]);
}

Output:
apple 6
berry 12
cherry 8

for...of does not work with objects!

Values, entries, and keys#

  • You can get just the values of an object with Object.values():

    console.log(Object.values(fruitsInventory));  // [6, 12, 8]

  • Object.entries() gets you both the key and the value:

    for (const [fruit, num] of Object.entries(fruits)) {
  console.log(fruit, num);
}

  • There is also Object.keys() which gets you an array of keys.

Caution: Don’t use for...in with arrays#

const fruits = ['apple', 'berry', 'cherry'];

for (const f in fruits) {
  console.log(f);
}
// Output:
// 0
// 1
// 2

If you try to loop through an array with for...in, you’ll actually get all the indices.

Copying and Destructuring#

Copying#

You can use the spread operator to copy both arrays and objects.

wb-objects-demo/spread.js#
const fruitsInventory = {
  apple: 6,
  berry: 12,
  cherry: 8,
};

const inventoryCopy = { ...fruitsInventory };
console.log(inventoryCopy); // { apple: 6, berry: 12, cherry: 8 }

const fruitsArray = ['apple', 'berry', 'cherry'];
const fruitsCopy = [...fruitsArray];
console.log(fruitsCopy); // [ 'apple', 'berry', 'cherry' ]

More possibilities with the spread operator

There’s lots more you can do with the spread operator. You can convert strings to arrays, merge two objects/arrays, or create a copy of an array/object that also contains additional data. There are some examples of this in your demo code.

Destructuring arrays#

We can “unpack” an array into different values using destructuring syntax:

const fruitsArray = ['apple', 'berry', 'cherry'];

const [fruit0, fruit1, fruit2] = fruitsArray;
console.log(fruit0); // apple
console.log(fruit1); // berry
console.log(fruit2); // cherry

Object.entries() is another example of destructuring arrays. Each entry is an array containing the key and the value.

const fruitsInventory = { apple: 6, berry: 12, cherry: 8 };

// Object.entries(fruitsInventory) contains 3 arrays:
// [['apple', 6], ['berry', 12], ['cherry', 8]]

for (const [fruit, num] of Object.entries(fruitsInventory)) {
  // each array gets unpacked into 2 values, 'fruit' and 'num'
  console.log(fruit, num);
}

Output:
apple 6
berry 12
cherry 8

Destructuring objects#

When destructuring objects, the variable names have to match the names of the keys. Also, you can leave some of them out.

wb-js-3-demo/destructuring.js#
const fruitsInventory = { apple: 6, berry: 12, cherry: 8 };

const { apple, cherry } = fruitsInventory;
console.log(apple); // 6
console.log(cherry); // 8

Nested data structures#

Nested data structures#

Arrays and objects can be nested inside each other:

wb-objects-demo/nested.js#
const fruits = [
  {
    name: 'apple',
    genus: 'Malus',
    colors: ['red', 'yellow', 'green'],
  },
  {
    name: 'berry',
    genus: 'Vaccinium',
    colors: ['red', 'blue'],
  },
  {
    name: 'cherry',
    genus: 'Prunus',
    colors: ['red'],
  },
];

console.log(Array.isArray(fruits)); // true
console.log(typeof fruits[0]); // Object
console.log(fruits[0].colors); // ['red', 'yellow', 'green']
console.log(fruits[0].colors[1]); // 'yellow'

To iterate through a nested structure like this, we can use a for...in loop inside a for...of loop.

const fruits = [
  {
    name: 'apple',
    genus: 'Malus',
    colors: ['red', 'yellow', 'green'],
  },
  {
    name: 'berry',
    genus: 'Vaccinium',
    colors: ['red', 'blue'],
  },
  {
    name: 'cherry',
    genus: 'Prunus',
    colors: ['red'],
  },
];

// Looping through an array of objects
for (const fruit of fruits) {
  // fruit is an object, loop through its keys
  for (const key in fruit) {
    console.log(key, fruit[key]);
  }
  console.log('');
}
